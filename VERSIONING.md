# Versioning

<!-- SPDX-License-Identifier: Apache-2.0 -->

## Source of truth

`version.json` is the only committed source for CapOwn product and control
plane protocol versions:

- `product_version` identifies the Master, Worker, MCP server, and release
  artifacts.
- `protocol_version` identifies the language-independent control-plane wire
  contract.

The values may be equal while the project is released as one coordinated
product. They remain separate fields because product and protocol releases may
advance independently later.

The MCP protocol version (`2025-03-26`), plugin versions, dependency versions,
and persisted `schema_version` values are independent version systems and must
not be changed with the CapOwn product version.

## Build and runtime rules

- Master runtime version values live in `master/internal/version` and are
  injected into release binaries with Go linker flags. A plain unversioned Go
  development build uses development defaults and is not a release artifact.
- Worker runtime values are generated by `node scripts/version.mjs sync-worker`
  into the ignored `worker/src/generated/` directory. Worker package metadata
  is synchronized from `version.json` by the same command.
- OpenAPI requires literal version strings. `protocol/openapi.yaml` is checked
  against `version.json` by `node scripts/version.mjs check`; do not edit its
  version fields to a different value.
- Docker builds read the manifest for the embedded Master runtime values. A
  release build must pass `CAPOWN_PRODUCT_VERSION` and
  `CAPOWN_PROTOCOL_VERSION` to Compose so the image tag and OCI labels match
  the manifest.

## Release procedure

1. Edit only the appropriate field in `version.json`.
2. Run `node scripts/version.mjs sync-worker`.
3. Run `node scripts/version.mjs check`.
4. Validate `protocol/openapi.yaml` with Swagger CLI.
5. Build the Master with `scripts/install-master.sh` or
   `scripts/install-master.ps1`, or build its container with the version
   variables exported from `version.json`:

   ```bash
   export CAPOWN_PRODUCT_VERSION="$(node scripts/version.mjs product)"
   export CAPOWN_PROTOCOL_VERSION="$(node scripts/version.mjs protocol)"
   docker compose -f master/compose.yaml build
   ```

   On PowerShell:

   ```powershell
   $env:CAPOWN_PRODUCT_VERSION = node scripts/version.mjs product
   $env:CAPOWN_PROTOCOL_VERSION = node scripts/version.mjs protocol
   docker compose -f master/compose.yaml build
   ```
6. Run the Master and Worker checks from `AGENTS.md`.

When a version cannot be referenced directly because a tool requires a literal
value, the value must be updated by the synchronization command or validated
by the consistency check before release. Do not introduce another manual
version source.
